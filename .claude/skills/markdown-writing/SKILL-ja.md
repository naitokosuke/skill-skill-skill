# Markdown 執筆ガイド

## 基本方針

- シンプルで読みやすい文書を心がける
- ファイルは UTF-8 で保存する
- 日本語文字が含まれる場合、文字化けに注意する
- 可能な限り簡潔に書く
- 冗長な表現を避ける

## 禁止事項

- 番号付きリスト（1. 2. 3.）を使わず `-` を使う
- 見出しに連番（`### 1. xxx`, `### 2. xxx` や `## 1. xxx` など）を付けない
  - 番号付きリスト禁止を見出しで回避する典型的なハックで、実質的に番号付きリストと同じ
  - 項目の追加・削除のたびに振り直しが必要でメンテが破綻する
  - 列挙したいだけなら箇条書き（`-`）に、独立セクションが必要なら内容を表す名前の見出しを付ける
- 段落ラベル（短い名前行 + 空行 + 値、の反復）を使わない
  - 例: `根拠\n\n<内容>\n\n説明\n\n<内容>\n\n提案\n\n<内容>` のような形
  - これは `**項目名**:` 禁止を改行で迂回するハックで、構造としては同じ key/value
  - 必要に応じて見出し（`###` など）にする、列挙なら箇条書き、それ以外は自然な散文で書く
- `**項目名**:` のような Markdown にない構造化記法を使わない
- 箇条書きで「キー: 値」を表すコロン区切り表記は絶対に使わない
  - `- 名前: 山田太郎` のような書き方は不可
  - 情報は自然な文として書く、または見出しと段落で構造化する
  - これは `**項目名**:` と同様に Markdown 本来の記法ではないため、一切使用禁止
- 表（テーブル）を使わない
- 強調（`**太字**`）を使わない
- 区切り線（`---`）を使わない
- 句点「。」を使わない
  - 文末は改行で表現する

悪い例（`**項目名**:` 構造）

```markdown
**名前**: 山田太郎
**役職**: エンジニア
```

悪い例（コロン区切り箇条書き — これも絶対に書かない）

```markdown
- 名前: 山田太郎
- 役職: エンジニア
```

良い例（自然な文で書く）

```markdown
山田太郎はエンジニア
```

良い例（見出しと段落で構造化する）

```markdown
## 山田太郎

エンジニア
```

## 日本語のルール

- 日本語と英数字の間には半角スペースを入れる
  - 良い例: `Claude は AI です`
  - 悪い例: `Claudeは AIです`
- 読点は「、」を使う
- 括弧は全角「（）」または半角「()」で統一する

## 文書構造

- 見出しは `#` から順に使い、階層を飛ばさない
- 最上位の見出し `#` は文書に 1 つだけ使う
- 段落間は空行を 1 行入れる
- リストの前後にも空行を入れる
- 文章は句点で繋がず改行で区切る

```markdown
# タイトル

概要文

## セクション

本文

- 項目 A
- 項目 B

次の段落
```

## コードブロック

- インラインコードは `` ` `` で囲む
- 複数行のコードは言語を指定してフェンスで囲む

````markdown
`変数名` を使う

```python
def hello():
    print("Hello")
```
````

## リンクと画像

```markdown
[リンクテキスト](URL)
![代替テキスト](画像パス)
```
